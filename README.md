# 2026 개발자 성장 기록

## 📌 이 레포지토리를 만든 이유
그동안 여러 주제를 공부하며 많은 것을 시작했지만, 명확한 기준과 기록이 없어 끝까지 이어가지 못한 경우가 많았다.<br/>
이 레포지토리는 산발적인 학습을 하나의 흐름으로 만들기 위한 시도이며, 시작보다 완주를 중요하게 생각하는 개발자가 되기 위한 기록이다. 

## 🛠️ 학습 원칙
- 개인적인 선호나 감에 의존해 중요도를 판단하지 않는다.
- 학습한 주제마다 직접 아이디어를 도출해 예제 코드와 정리 글을 작성한다.
- 학습을 시작하기 전에 “왜 이걸 공부하는가”에 대한 답을 먼저 정리한다.
